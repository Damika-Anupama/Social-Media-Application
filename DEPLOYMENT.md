# Deployment — Pulse (frontend-demo branch)

This document captures how the `frontend-demo` branch is deployed to Vercel.

## ⚠️ The live demo is currently private

**URL:** https://social-media-application-damika-anupamas-projects.vercel.app

Vercel **Deployment Protection** is switched on, so every visitor is redirected
to a Vercel login page instead of the app. The deploys themselves are healthy —
the site is simply not reachable by anyone but the account owner, which for a
portfolio demo is the same as being down.

**Fix (dashboard only — it cannot be changed from the repo):**

> Vercel → Project → Settings → **Deployment Protection** →
> **Vercel Authentication: Disabled** → Save

Then confirm it is genuinely public:

```bash
E2E_BASE_URL=https://social-media-application-damika-anupamas-projects.vercel.app npm run test:smoke
```

The smoke suite fails with an explicit message while the auth wall is up. Note
that nothing else can catch this: the build, the unit tests, the 548 e2e tests
and CI all run against a *local* build, and were green the entire time the live
site was serving a login screen.

## Metadata / canonical URLs

`metadataBase` is derived from the deployment environment
(`NEXT_PUBLIC_SITE_URL` → `VERCEL_PROJECT_PRODUCTION_URL` → `VERCEL_URL` →
localhost). It used to be hardcoded to `pulse-demo.vercel.app`, a domain owned
by somebody else — so every canonical link, OpenGraph tag and share preview
advertised a stranger's site. Set `NEXT_PUBLIC_SITE_URL` if a custom domain is
ever added; otherwise it resolves correctly on its own.

## Production branch

**`frontend-demo`** is the production branch. After importing into Vercel, set
this explicitly:

> Vercel dashboard → Settings → Git → **Production Branch: `frontend-demo`**

If you skip this, pushes to `frontend-demo` will only generate Preview
deployments and the production URL will not update.

## One-time setup (Vercel dashboard)

1. Open https://vercel.com/new and import the GitHub repo
   `Damika-Anupama/Social-Media-Application`.
2. **Framework preset:** Next.js (auto-detected).
3. **Root Directory:** repo root (default — the `frontend-demo` branch has the
   Next.js app at root).
4. **Build settings:** loaded from the committed `vercel.json` —
   - Build command: `npm run build`
   - Output directory: `.next`
   - Install command: `npm install --legacy-peer-deps`
5. **Environment variables:** none required. The app uses only mocked data.
6. Click **Deploy**.
7. After the first deploy: **Settings → Git → Production Branch →
   `frontend-demo`**.

## Redeploys

`git push origin frontend-demo` — Vercel rebuilds and promotes automatically
because `frontend-demo` is set as the production branch.

## Verify checklist after each deploy

Click through the golden path on the production URL:

- `/` — landing page loads, the hero animates, no console errors.
- `/login` — submit with invalid input (empty, malformed email, weak password) —
  inline errors appear. Submit with a valid email + password (e.g.
  `user@example.com` / `Pulse2026`) — you should land on `/dashboard`.
- `/register` — same as above, plus username rules, password strength meter,
  confirm-match check, and the terms checkbox. Valid input lands on
  `/dashboard`.
- `/dashboard` — feed renders, story rail scrolls, post composer accepts text,
  likes / re-shares / bookmarks toggle.
- `/dashboard/messages` — sidebar list shows, sending a message appends to the
  thread.
- `/dashboard/explore`, `/notifications`, `/profile`, `/settings`,
  `/bookmarks`, `/communities` — all render without errors.
- Mobile viewport (iPhone 12) — bottom tab bar replaces the sidebar; no
  horizontal scroll.

## Rollback

If a deploy ships a regression:

- **Dashboard:** Deployments → previous green deployment → ⋯ → Promote to
  Production. Instant rollback, no rebuild.
- **CLI** (if `vercel` is installed): `vercel rollback <deployment-url>`.

## Notes

- This is a frontend demonstration. Any input that passes client-side
  validation on the auth pages is accepted as a "login" and redirects to
  `/dashboard`. There is no backend.
- All photography is sourced from Unsplash CDN; avatars are generated by
  DiceBear. Both hostnames are allowlisted in `next.config.js`.
- The `main` branch keeps a slim snapshot of the original Angular 11 +
  NativeScript frontend and Spring Boot backend for historical reference. It is
  **not** deployable — do not point Vercel at it.
