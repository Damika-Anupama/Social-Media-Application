# Deployment ‚Äî Pulse (frontend-demo branch)

This document captures how the `frontend-demo` branch is deployed to Vercel.

## Production branch

**`frontend-demo`** is the production branch. After importing into Vercel, set
this explicitly:

> Vercel dashboard ‚Üí Settings ‚Üí Git ‚Üí **Production Branch: `frontend-demo`**

If you skip this, pushes to `frontend-demo` will only generate Preview
deployments and the production URL will not update.

## One-time setup (Vercel dashboard)

1. Open https://vercel.com/new and import the GitHub repo
   `Damika-Anupama/Social-Media-Application`.
2. **Framework preset:** Next.js (auto-detected).
3. **Root Directory:** repo root (default ‚Äî the `frontend-demo` branch has the
   Next.js app at root).
4. **Build settings:** loaded from the committed `vercel.json` ‚Äî
   - Build command: `npm run build`
   - Output directory: `.next`
   - Install command: `npm install --legacy-peer-deps`
5. **Environment variables:** none required. The app uses only mocked data.
6. Click **Deploy**.
7. After the first deploy: **Settings ‚Üí Git ‚Üí Production Branch ‚Üí
   `frontend-demo`**.

## Redeploys

`git push origin frontend-demo` ‚Äî Vercel rebuilds and promotes automatically
because `frontend-demo` is set as the production branch.

## Verify checklist after each deploy

Click through the golden path on the production URL:

- `/` ‚Äî landing page loads, the hero animates, no console errors.
- `/login` ‚Äî submit with invalid input (empty, malformed email, weak password) ‚Äî
  inline errors appear. Submit with a valid email + password (e.g.
  `user@example.com` / `Pulse2026`) ‚Äî you should land on `/dashboard`.
- `/register` ‚Äî same as above, plus username rules, password strength meter,
  confirm-match check, and the terms checkbox. Valid input lands on
  `/dashboard`.
- `/dashboard` ‚Äî feed renders, story rail scrolls, post composer accepts text,
  likes / re-shares / bookmarks toggle.
- `/dashboard/messages` ‚Äî sidebar list shows, sending a message appends to the
  thread.
- `/dashboard/explore`, `/notifications`, `/profile`, `/settings`,
  `/bookmarks`, `/communities` ‚Äî all render without errors.
- Mobile viewport (iPhone 12) ‚Äî bottom tab bar replaces the sidebar; no
  horizontal scroll.

## Rollback

If a deploy ships a regression:

- **Dashboard:** Deployments ‚Üí previous green deployment ‚Üí ‚ãØ ‚Üí Promote to
  Production. Instant rollback, no rebuild.
- **CLI** (if `vercel` is installed): `vercel rollback <deployment-url>`.

## Notes

- This is a frontend demonstration. Any input that passes client-side
  validation on the auth pages is accepted as a "login" and redirects to
  `/dashboard`. There is no backend.
- All photography is sourced from Unsplash CDN; avatars are generated by
  DiceBear. Both hostnames are allowlisted in `next.config.js`.
- The `main` branch keeps a slim snapshot of the original Angular 11 +
  NativeScript frontend and Spring Boot backend for historical reference. It is
  **not** deployable ‚Äî do not point Vercel at it.
